Уровень 3. Сложные / продвинутые боты
Примеры:
Платежные системы (оплата через Telegram Payments, подписки).
Маркетплейсы/магазины внутри бота.
Боты с личными кабинетами, аналитикой, статистикой.
Функциональность:
Много состояний, сложные воронки.
Платежи, баланс пользователей, реферальные программы.
Мультиязычность.
Инфраструктура:
Несколько сервисов (авторизация, биллинг, аналитика).
Redis для кеша, очереди (RabbitMQ, NATS, Kafka).
CI/CD, докеризация, мониторинг.
Заработок:
Серьезный чек, долгие проекты (недели–месяцы).
Долгосрочная поддержка/ретейнер.

## Реализация в telegram-bot-pooling-hard

Сделано в коде (MVP под этот шаблон):

- **Личный кабинет:** баланс в демо-кредитах (копейки), реферальный код и ссылка `t.me/<bot>?start=ref_<код>` (параметр `/start` обрабатывается при входе).
- **Платежи:** списание фиксированной суммы с баланса при подтверждении записи к врачу (не Telegram Payments; задел под биллинг).
- **Реферальная программа:** приглашение по коду; после завершения регистрации (ФИО+телефон) бонусы начисляются приглашённому и пригласившему.
- **Мультиязычность:** RU/EN, кнопки «EN»/«RU», учёт `language_code` из Telegram и поля `preferred_lang` в профиле.
- **Аналитика:** таблица `analytics_events`, события `cmd_start`, `booking_confirmed`; в админке — сводка за 7 дней.
- **Redis:** опционально (`REDIS_ADDR`), кеш страниц списка специализаций (TTL 30 с); в Compose по умолчанию `redis:6379`.
- **CI:** `.github/workflows/ci.yml` — тесты, vet, сборка Docker-образа.

Не входит в объём (можно развивать дальше): отдельный сервис биллинга, очереди (RabbitMQ/NATS), Telegram Payments/Stars, продвинутый мониторинг.
